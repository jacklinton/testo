    • This repository is designed to be a base configuration for all Node.js based micro services using the LTS version of Node.js with is currently around 12.13.1.

    In its original state, it runs without any build process with Babel, Typescript, or anything else. The code simply runs as it is. However, the Typescript language service is still utilized so that the developer can take advantage of type checking without writing actual typescript. The option to write typescript is also there. Typescript will also generate type definition files for *.js files based on inference and whatever is explicitly defined via JSDoc, ideally as part of the CI process (mostly for reusable code that will be registered in package registry).

    One of the primary goals of this project is to create a boilerplate that has as little bulk as possible so that it can be used with Alpine docker images and still be relatively small (I think the base image for Node.js is 25Mb). That means using as few dependencies as possible and utilizing built-in javascript functionality over 3rd party modules. Another goal is to have everything setup so that when a dev creates a new project using this boilerplate, all they have to do is create their route/s files and tests for those files and probably make some changes to config. The linter and test suite will eventually enforce certain standards before changes can be committed or merged to reduce failures at the CI level.

    The project will use PM2 as a process manager to spawn a single fork of the application and thus one process, not a cluster. More research needs to be done whether PM2 is sufficient for development, however it is certainly the best tool for production. I will leave nodemon on the table for development unless a way is found to replicate it functionality with PM2 (likely).
